Problems with Dashcode
meat-eater.blogspot.com—apple-dashcode-widget-creation-for-all.html
“ From the looks of it, this app is not a consumer oriented app for widget creation. It looks like it will be an IDE for dashboard widgets with very nice WYSIWYG tools. Even the name, which is reminiscent of Xcode, hints at this. Not that it will be a bad thing, but it is not what I first thought it will be. I am confident that there a lot of end users (read as: non-programmers) who have their own ideas for really simple but specialized dashboard widgets. They will benefit greatly from a consumer oriented widget creation app. Sadly, that app won't be dashcode. But the sadder thing is that this mythical consumer oriented widget creating app that I desire, seems nowhere in sight. ”
Dashboard Dev : Info.plist Tutorial
dashdev.djupet.se—dashboard_widget_tutorial_1.html
“ There are 5 properties that should be included in a widget.
1. CFBundleDisplayName - The widget display-name
2. CFBundleIdentifier - The widget identifier
3. CFBundleName - The widget name
4. CFBundleVersion - The widget version
5. MainHTML - A link to the html file it should load on start, if you are into web, you can think of the file as index.html ”
App Update Widget
www.skia.net—dev.php
“ Derived from the successful Widget Update, App Update enables you to check for updates to your share- and freeware applications at Apple, MacUpdate and VersionTracker. You will be presented with a tidy list of links to the detail pages of all the updates found. Among the key features are an extensive online-help, a configuration editor to tailor App Update to your very personal collection of applications, limited support for Quicktime Components, Frameworks and Preference Panes, automatic checking for updates and Growl support. Like for Widget Update, graphics were designed by Mike Pogran. ”
